# คู่มือการติดตั้งและอัปโหลดแอปพลิเคชันขึ้น GitHub Pages (ฉบับสมบูรณ์)

คู่มือนี้รวบรวมขั้นตอน วิธีการแก้ไขปัญหา และโครงสร้างไฟล์ที่จำเป็นเพื่อให้แอปพลิเคชันของคุณทำงานบน GitHub Pages ได้อย่างถูกต้อง

---

## 📋 สรุปปัญหาที่พบและวิธีแก้ไข
1. **ปัญหา Error 404 (MIME type / main.tsx):**
   - **สาเหตุ:** GitHub Pages ไม่สามารถรันไฟล์ `.tsx` ได้โดยตรง และมักจะหาไฟล์ไม่เจอหากโครงสร้างโฟลเดอร์ซับซ้อนเกินไป
   - **วิธีแก้ไข:** ทำการแปลงไฟล์ (Build) ให้เป็นไฟล์ Static (`.js`, `.css`) และจัดวางไว้ที่หน้าแรก (Root) ของ Repository

2. **ปัญหาชื่อไฟล์สุ่ม (Hashing):**
   - **สาเหตุ:** ระบบเดิมจะสุ่มชื่อไฟล์ทุกครั้งที่ Build (เช่น `index-f9Sjq9BL.js`) ทำให้ยากต่อการอัปเดต
   - **วิธีแก้ไข:** ปรับแต่งระบบ Build ให้ใช้ชื่อคงที่คือ `index.js` และ `index.css` เพื่อให้ง่ายต่อการจัดการ

3. **ปัญหาไฟล์ .nojekyll หาย:**
   - **สาเหตุ:** ไฟล์ที่ขึ้นต้นด้วยจุดมักถูกซ่อนโดยระบบปฏิบัติการ
   - **วิธีแก้ไข:** สร้างไฟล์นี้โดยตรงบน GitHub หรือรวบรวมไว้ในโฟลเดอร์ที่ชัดเจน

---

## 📂 โครงสร้างไฟล์ที่ต้องใช้ (ในโฟลเดอร์ `GITHUB_FILES`)
เพื่อให้แอปทำงานได้ คุณต้องอัปโหลดไฟล์ 4 ไฟล์นี้ขึ้นไปที่ GitHub:
1. **index.html**: ไฟล์หลักที่เบราว์เซอร์จะเรียกอ่าน
2. **index.js**: ไฟล์โค้ดการทำงานทั้งหมด (ที่ถูกแปลงมาจาก React/TSX)
3. **index.css**: ไฟล์การตกแต่งหน้าตาแอป
4. **.nojekyll**: ไฟล์เปล่าที่บอกให้ GitHub ไม่ต้องใช้ระบบ Jekyll (สำคัญมากสำหรับการโหลดไฟล์ JS/CSS)

---

## 🚀 ขั้นตอนการอัปโหลดขึ้น GitHub (Step-by-Step)

### 1. เตรียมพื้นที่บน GitHub
* เข้าไปที่ Repository ของคุณบน GitHub
* **ลบไฟล์เก่าออกให้หมด** (ยกเว้น README.md) เพื่อไม่ให้ไฟล์เก่าไปตีกับไฟล์ใหม่

### 2. ดาวน์โหลดไฟล์จากระบบนี้
* ไปที่เมนู **Settings** (รูปเฟือง) -> **Export** -> **Download ZIP**
* แตกไฟล์ ZIP ที่ดาวน์โหลดมาลงในเครื่องคอมพิวเตอร์ของคุณ

### 3. อัปโหลดไฟล์
* เข้าไปในโฟลเดอร์ **`GITHUB_FILES`**
* ลากไฟล์ทั้ง 4 ไฟล์ (`index.html`, `index.js`, `index.css`, `.nojekyll`) ขึ้นไปที่หน้าแรก (Root) ของ GitHub Repository
* กด **Commit changes** เพื่อยืนยันการอัปโหลด

### 4. วิธีสร้างไฟล์ .nojekyll บน GitHub (กรณีหาไฟล์ไม่เจอ)
หากในเครื่องของคุณมองไม่เห็นไฟล์ `.nojekyll` ให้ทำดังนี้:
* ในหน้า Repository บน GitHub กดปุ่ม **Add file** -> **Create new file**
* ตั้งชื่อไฟล์ว่า `.nojekyll` (ต้องมีจุดนำหน้า)
* **ไม่ต้องพิมพ์อะไรลงไป** ปล่อยให้ว่างไว้ แล้วกด **Commit changes**

## 🛠️ การตรวจสอบเมื่อพบปัญหา
* **รอการอัปเดต:** หลังจากอัปโหลด GitHub ต้องใช้เวลา 1-2 นาทีในการประมวลผล
* **ล้างแคช:** หากหน้าเว็บไม่เปลี่ยน ให้กด `Ctrl + F5` (Windows) หรือ `Cmd + Shift + R` (Mac)
* **ตรวจสอบตำแหน่งไฟล์:** ไฟล์ทั้ง 4 ต้องอยู่ที่หน้าแรกสุดของ Repository เท่านั้น ห้ามอยู่ในโฟลเดอร์ย่อย
